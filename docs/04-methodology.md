# Methodology - doc-organizer

## Objetivo
Definir como o desenvolvimento do doc-organizer será organizado e executado ao longo do tempo.

---

# 1. Modelo de desenvolvimento

O projeto será desenvolvido de forma incremental e individual, seguindo um fluxo baseado em Kanban pessoal.

Não será utilizado Scrum formal ou sprints rígidos nesta fase.

---

# 2. Organização do trabalho

O desenvolvimento será dividido em três estados principais:

## To Do
Tarefas que ainda não foram iniciadas.

## Doing
Tarefas que estão sendo desenvolvidas no momento.

## Done
Tarefas já concluídas.

---

# 3. Fonte de organização do projeto

O arquivo `06-backlog.md` será utilizado como principal fonte de controle das tarefas do projeto.

Todas as funcionalidades e melhorias serão registradas e movidas entre estados conforme o progresso.

---

# 4. Fluxo de desenvolvimento

O fluxo básico será:

1. Definir uma tarefa no backlog
2. Mover tarefa para "Doing"
3. Implementar funcionalidade no código
4. Atualizar documentação se necessário
5. Mover tarefa para "Done"

---

# 5. Abordagem de desenvolvimento

O desenvolvimento seguirá os princípios:

- Construir primeiro o MVP funcional
- Evoluir o sistema de forma incremental
- Evitar complexidade desnecessária no início
- Priorizar funcionamento antes de otimização

---

# 6. Controle de evolução do projeto

O projeto será evoluído em fases (temporadas conceituais), onde cada fase representa um conjunto de funcionalidades entregues.

As fases serão registradas na documentação, sem necessidade de duplicação de arquivos.
